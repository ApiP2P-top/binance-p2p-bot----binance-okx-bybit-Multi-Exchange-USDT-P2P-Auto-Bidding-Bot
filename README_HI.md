## 📝 अवलोकन और आर्किटेक्चर

**binance-p2p-bot** एक पेशेवर-स्तर का डेस्कटॉप क्लाइंट है जो **Binance**, **Bybit**, और **OKX** पर कार्यरत **P2P व्यापारियों (विज्ञापनदाताओं)** के लिए डिज़ाइन किया गया है। मैन्युअल बाज़ार निगरानी से API-संचालित प्रोग्रामेटिक निष्पादन में बदलकर, यह USDT P2P बॉट आपको तीनों प्रमुख एक्सचेंजों पर एक साथ लगातार मूल्य लाभ और कुशल ऑर्डर प्रोसेसिंग गति बनाए रखने में मदद करता है।

**मॉड्यूलर इवेंट-ड्रिवन आर्किटेक्चर** और **शेड्यूल्ड API पोलिंग तथा कॉन्फ़िगर करने योग्य इंटरवल** पर निर्मित, यह मैन्युअल ट्रैकिंग में देरी, आर्बिट्राजर्स को राजस्व हानि, और थकाऊ "स्क्रीन ट्रैप" की समस्या को संबोधित करता है। `binance-p2p-bot` इंजन वर्तमान बाज़ार स्थितियों के आधार पर पोलिंग फ़्रीक्वेंसी को समायोजित करने के लिए **अनुकूली इंटरवल शेड्यूलिंग** का उपयोग करता है।

## 🚀 मुख्य विशेषताएं (एल्गोरिदम और वर्कफ़्लो)

### 1. ऑटो-बिडिंग और रियल-टाइम मूल्य समायोजन (स्मार्ट रैंक स्नाइपिंग)

* **शेड्यूल्ड मार्केट पोलिंग:** `binance-p2p-bot` इंजन प्रत्येक एक्सचेंज की P2P विज्ञापन सूची के खिलाफ़ समय-समय पर REST API क्वेरी करता है, प्रत्येक पोलिंग चक्र में प्रतिस्पर्धी मूल्य परिवर्तनों का पता लगाता है।
* **वेटेड प्रायोरिटी प्राइसिंग:** एक **वेटेड स्कोरिंग एल्गोरिदम** का उपयोग करता है जो प्रतिस्पर्धी कीमतों का मूल्यांकन करता है, कॉन्फ़िगर करने योग्य ऑफ़सेट मान (जैसे, $0.01 एज) लागू करता है, और अग्रणी विज्ञापन स्थिति बनाए रखने के लिए एक इष्टतम काउंटर-प्राइस की गणना करता है।
* **मल्टी-क्राइटेरिया प्रतिस्पर्धी फ़िल्टरिंग:** एक **कॉन्फ़िगर करने योग्य फ़िल्टर पाइपलाइन** तैनात करता है — इंजन व्यापार राशि सीमाओं, न्यूनतम ऑर्डर लिमिट्स, भुगतान विधि संगतता, और ऑनलाइन स्थिति के आधार पर प्रतिस्पर्धी विज्ञापनों का मूल्यांकन करता है।
* **सुरक्षित रेंज सुरक्षा:** अत्यधिक फ़िएट/क्रिप्टो अस्थिरता के दौरान अलाभकारी लेनदेन को रोकने के लिए **कॉन्फ़िगर करने योग्य सीमा नियमों** के माध्यम से सख्त मूल्य सीलिंग और फ़्लोर लागू करता है।

#### 💡 कार्यान्वयन अवधारणा (स्मार्ट स्नाइपिंग)
```python
# [स्यूडोकोड] वेटेड प्रायोरिटी प्राइसिंग इंजन
class PricingEngine:
    def __init__(self, exchange_adapter, config):
        self.adapter = exchange_adapter
        self.config = config
        self.poll_interval_ms = 2000
    
    async def on_poll_cycle(self):
        """शेड्यूल्ड पोलिंग टाइमर द्वारा ट्रिगर"""
        # REST API के माध्यम से वर्तमान प्रतिस्पर्धी विज्ञापन सूची प्राप्त करें
        competitor_ads = await self.adapter.search_ads(
            asset=self.config.asset,
            fiat=self.config.fiat,
            trade_type=self.config.trade_type
        )
        
        # मल्टी-क्राइटेरिया फ़िल्टर पाइपलाइन लागू करें
        candidates = self.filter_pipeline.execute(
            competitor_ads,
            filters=self.config.active_filters
        )
        
        # वेटेड स्कोरिंग के माध्यम से लक्ष्य मूल्य की गणना करें
        target_price = self.scoring_engine.calculate(
            candidates,
            offset=self.config.price_offset,
            floor=self.config.min_price,
            ceiling=self.config.max_price
        )
        
        # यदि बदलाव हो तो मूल्य अपडेट निष्पादित करें
        if abs(target_price - self.current_price) > self.config.epsilon:
            await self.adapter.update_ad_price(self.ad_id, target_price)
            self.current_price = target_price
```

### 2. मल्टी-एक्सचेंज एकीकृत नियंत्रण (Binance + Bybit + OKX)

* **थ्रेड-आइसोलेटेड एक्सचेंज मॉड्यूल:** प्रत्येक एक्सचेंज (Binance, Bybit, OKX) अपने स्वयं के **स्वतंत्र थ्रेड पूल** में अलग शेड्यूलिंग के साथ चलता है, क्रॉस-एक्सचेंज हस्तक्षेप को रोकता है।
* **एब्स्ट्रैक्ट एडैप्टर पैटर्न:** एक एकीकृत `ExchangeAdapterFactory` एक **फ़ैक्टरी पैटर्न** के माध्यम से प्लेटफ़ॉर्म-विशिष्ट एडैप्टर बनाता है, तीनों एक्सचेंजों में API अंतरों को पारदर्शी रूप से सामान्य करता है।
* **OKX रेट-लिमिट कतार:** OKX की सख्त 5-कॉल/मिनट सीमा को **टाइमर-आधारित स्पेसिंग वाली डिबाउंस कतार** द्वारा संभाला जाता है, जो सीमा के भीतर रहने के लिए समवर्ती अपडेट अनुरोधों को स्वचालित रूप से जोड़ती है।

#### 💡 कार्यान्वयन अवधारणा (मल्टी-एक्सचेंज एडैप्टर)
```python
# [स्यूडोकोड] थ्रेड-आइसोलेटेड एक्सचेंज एडैप्टर फ़ैक्टरी
class ExchangeAdapterFactory:
    _registry = {}
    
    @classmethod
    def register(cls, platform: str, adapter_cls):
        cls._registry[platform] = adapter_cls
    
    @classmethod
    def create(cls, platform: str, credentials: dict):
        adapter_cls = cls._registry.get(platform)
        if not adapter_cls:
            raise UnsupportedPlatformError(platform)
        
        # प्रत्येक एडैप्टर अपने स्वयं के थ्रेड पूल में चलता है
        thread_pool = ThreadPoolExecutor(
            max_workers=platform_config(platform).max_concurrency
        )
        return adapter_cls(credentials, executor=thread_pool)

# सभी समर्थित एक्सचेंजों को पंजीकृत करें
ExchangeAdapterFactory.register("binance", BinanceP2PAdapter)
ExchangeAdapterFactory.register("bybit", BybitP2PAdapter)
ExchangeAdapterFactory.register("okx", OkxP2PAdapter)
```

### 3. स्वचालित इन्वेंट्री पाइपलाइन और ऑटो-रिप्लेनिशमेंट

* **मल्टी-स्टेप एसेट कन्वर्ज़न:** `binance-p2p-bot` फ़ंडिंग अकाउंट बैलेंस की निगरानी करता है और स्वचालित रूप से एक **अनुक्रमिक कन्वर्ज़न पाइपलाइन** (USDC/FDUSD → स्पॉट मार्केट सेल → USDT → फ़ंडिंग अकाउंट) ट्रिगर करता है, जिससे आपकी USDT इन्वेंट्री भरी रहती है।
* **थ्रेशोल्ड-आधारित निगरानी:** यह **कॉन्फ़िगर करने योग्य बैलेंस थ्रेशोल्ड** का उपयोग करता है ताकि यह पता लगाया जा सके कि परिवर्तनीय एसेट न्यूनतम राशि से अधिक हो गई है, केवल तभी कन्वर्ज़न चक्र ट्रिगर करता है जब सार्थक बैलेंस उपलब्ध हो।
* **बैच इन्वेंट्री सिंक्रनाइज़ेशन:** कन्वर्ज़न के बाद, बॉट उपलब्ध USDT की पुनर्गणना करता है और एक ही बैच ऑपरेशन में सभी सक्रिय विज्ञापनों पर **अपडेटेड इन्वेंट्री प्रसारित** करता है, जिससे सभी लिस्टिंग में सुसंगत मात्रा सुनिश्चित होती है।

#### 💡 कार्यान्वयन अवधारणा (इन्वेंट्री पाइपलाइन)
```python
# [स्यूडोकोड] मल्टी-स्टेप एसेट कन्वर्ज़न पाइपलाइन
class InventoryPipeline:
    CONVERSION_ROUTES = {
        "USDC": ["transfer_to_spot", "market_sell_for_usdt", "transfer_to_funding"],
        "FDUSD": ["transfer_to_spot", "market_sell_for_usdt", "transfer_to_funding"],
    }
    
    async def run_replenishment_cycle(self):
        """थ्रेशोल्ड-ट्रिगर्ड इन्वेंट्री रिप्लेनिशमेंट"""
        # 1. फ़ंडिंग अकाउंट में परिवर्तनीय एसेट्स की जांच करें
        balances = await self.exchange.get_funding_balances()
        
        for asset, amount in balances.items():
            if asset in self.CONVERSION_ROUTES and amount >= self.min_threshold:
                # 2. अनुक्रमिक कन्वर्ज़न चरण निष्पादित करें
                route = self.CONVERSION_ROUTES[asset]
                for step in route:
                    await self.execute_step(step, asset, amount)
        
        # 3. अंतिम USDT बैलेंस की पुनर्गणना करें
        usdt_available = await self.exchange.get_funding_balance("USDT")
        
        # 4. सभी सक्रिय विज्ञापनों पर अपडेटेड इन्वेंट्री प्रसारित करें
        for ad in self.active_ads:
            current = await self.adapter.get_ad_detail(ad.id)
            new_inventory = self.calculate_new_inventory(current, usdt_available)
            if abs(new_inventory - current.inventory) >= 1.0:
                await self.adapter.update_ad_inventory(ad.id, new_inventory)
```

### 4. प्रतिस्पर्धी विश्लेषण और ब्लैकलिस्ट इंजन

* **मल्टी-स्टेज फ़िल्टर पाइपलाइन:** इंजन प्रतिस्पर्धी विज्ञापनों पर एक **अनुक्रमिक फ़िल्टर श्रृंखला** लागू करता है — मूल्य रेंज, व्यापार राशि सीमाएं, भुगतान विधि ओवरलैप, ऑनलाइन स्थिति, और पहचान जांच का मूल्यांकन करके एक परिष्कृत उम्मीदवार सूची तैयार करता है।
* **कॉन्फ़िगर करने योग्य ब्लैकलिस्ट:** `binance-p2p-bot` एक **प्रति-विज्ञापन ब्लैकलिस्ट** का समर्थन करता है जहाँ व्यापारी विशिष्ट प्रतिस्पर्धी विज्ञापन नंबरों को बाहर कर सकते हैं। ब्लैकलिस्टेड विज्ञापनों को प्रत्येक बिडिंग चक्र में स्वचालित रूप से छोड़ दिया जाता है।
* **भुगतान विधि मैचिंग:** भुगतान विधि सरणियों पर **सेट-इंटरसेक्शन लॉजिक** का उपयोग करता है ताकि यह सुनिश्चित किया जा सके कि बॉट केवल उन विज्ञापनदाताओं के विरुद्ध प्रतिस्पर्धा करे जो आपके विज्ञापन के साथ कम से कम एक सामान्य भुगतान चैनल साझा करते हैं।

#### 💡 कार्यान्वयन अवधारणा (प्रतिस्पर्धी फ़िल्टर)
```python
# [स्यूडोकोड] मल्टी-स्टेज प्रतिस्पर्धी फ़िल्टर पाइपलाइन
class CompetitorFilterPipeline:
    def __init__(self, config: AdConfig):
        self.config = config
        self.filters = [
            self.filter_self,
            self.filter_blacklist,
            self.filter_price_range,
            self.filter_trade_amount,
            self.filter_min_order,
            self.filter_payment_methods,
            self.filter_online_status,
        ]
    
    def execute(self, raw_ads: list) -> list:
        """फ़िल्टर श्रृंखला को अनुक्रमिक रूप से लागू करें"""
        candidates = raw_ads
        for filter_fn in self.filters:
            candidates = [ad for ad in candidates if filter_fn(ad)]
        return sorted(candidates, key=lambda ad: ad.price)
    
    def filter_blacklist(self, ad) -> bool:
        return ad.ad_id not in self.config.blacklist
    
    def filter_payment_methods(self, ad) -> bool:
        """सेट-इंटरसेक्शन: कम से कम एक सामान्य भुगतान विधि"""
        my_methods = set(self.config.payment_methods)
        their_methods = set(ad.payment_methods)
        return len(my_methods & their_methods) > 0
    
    def filter_trade_amount(self, ad) -> bool:
        if self.config.max_amount_filter:
            return ad.max_amount >= self.config.max_amount_filter
        return True
```

### 5. विज्ञापन जीवनचक्र प्रबंधन और बल्क ऑपरेशन

* **स्टेटस ट्रांज़िशन कंट्रोल:** प्रत्येक विज्ञापन सीधे API कॉल के माध्यम से स्टेटस ट्रांज़िशन (`OFFLINE ↔ ONLINE`) का समर्थन करता है, अमान्य ऑपरेशन को रोकने के लिए स्टेटस वैलिडेशन के साथ।
* **रेट-लिमिट स्पेसिंग के साथ बल्क टॉगल:** बर्स्ट-ट्रिगर्ड रेट लिमिट को रोकने के लिए कॉन्फ़िगर करने योग्य API स्पेसिंग के साथ सभी विज्ञापनों के लिए एक-क्लिक सक्षम/अक्षम।
* **वेटेड एवरेज प्राइस ट्रैकिंग:** C2C ऑर्डर इतिहास से गणना किए गए वेटेड एवरेज खरीद/बिक्री मूल्यों सहित लाइव मेट्रिक्स, जो व्यापारियों को रियल-टाइम लाभप्रदता अंतर्दृष्टि प्रदान करता है।

#### 💡 कार्यान्वयन अवधारणा (विज्ञापन स्टेटस प्रबंधक)
```python
# [स्यूडोकोड] बल्क ऑपरेशन के साथ विज्ञापन स्टेटस प्रबंधक
class AdStatusManager:
    async def set_status(self, ad_id: str, online: bool):
        """API के माध्यम से एकल विज्ञापन स्टेटस टॉगल करें"""
        current = await self.adapter.get_ad_detail(ad_id)
        target_online = online
        
        if current.is_online == target_online:
            return  # पहले से वांछित स्थिति में है
        
        await self.adapter.set_ad_status(ad_id, online=target_online)
        logger.info(f"विज्ञापन {ad_id}: {'ONLINE' if target_online else 'OFFLINE'}")
    
    async def bulk_toggle(self, ad_ids: list, online: bool):
        """API स्पेसिंग के साथ बैच स्टेटस परिवर्तन"""
        for ad_id in ad_ids:
            await self.set_status(ad_id, online)
            await asyncio.sleep(0.5)  # रेट-लिमिट स्पेसिंग
```

### 6. स्टेट पर्सिस्टेंस और कॉन्फ़िगरेशन रिकवरी

* **परिवर्तन पर ऑटो-सेव:** प्रत्येक पैरामीटर संशोधन एक स्थानीय JSON कॉन्फ़िगरेशन फ़ाइल में तत्काल लिखाव ट्रिगर करता है, जिससे सभी सेटिंग्स लगातार सहेजी जाती हैं।
* **मास्क्ड क्रेडेंशियल स्टोरेज:** API कुंजियाँ UI में **कैरेक्टर मास्किंग** के साथ प्रदर्शित होती हैं (केवल अंतिम कुछ अक्षर दिखाते हुए) और स्थानीय रूप से संग्रहीत होती हैं। बॉट API कुंजियों को कभी भी "Withdrawal" अनुमतियाँ नहीं देने की सलाह देता है।
* **रीस्टार्ट रिकवरी:** एप्लिकेशन रीस्टार्ट पर, बॉट सहेजी गई कॉन्फ़िगरेशन को पढ़ता है और सभी पैनलों, विज्ञापन पंक्तियों, मूल्य सीमाओं, और बिडिंग पैरामीटर को उनकी शटडाउन-पूर्व स्थिति में बहाल करता है — कोई मैन्युअल पुनः प्रविष्टि आवश्यक नहीं।

#### 💡 कार्यान्वयन अवधारणा (स्टेट पर्सिस्टेंस)
```python
# [स्यूडोकोड] JSON-आधारित ऑटो-सेव कॉन्फ़िगरेशन प्रबंधक
class ConfigurationManager:
    def __init__(self, config_path: str):
        self.config_path = config_path
        self._state = {}
    
    def save_state(self, ui_state: dict):
        """प्रत्येक पैरामीटर परिवर्तन पर एटॉमिक राइट"""
        data = {
            "api_key": ui_state.get("api_key"),
            "api_secret": ui_state.get("api_secret"),
            "language": ui_state.get("language", "en"),
            "sell_ads": ui_state.get("sell_ads", []),
            "buy_ads": ui_state.get("buy_ads", []),
            "replenish_enabled": ui_state.get("replenish_enabled", False),
        }
        # अस्थायी फ़ाइल + रीनेम के माध्यम से एटॉमिक राइट
        tmp_path = self.config_path + ".tmp"
        with open(tmp_path, 'w', encoding='utf-8') as f:
            json.dump(data, f, indent=2, ensure_ascii=False)
        os.replace(tmp_path, self.config_path)
    
    def load_state(self) -> dict:
        """डिस्क से पूर्ण कॉन्फ़िगरेशन बहाल करें"""
        if not os.path.exists(self.config_path):
            return self._default_config()
        with open(self.config_path, 'r', encoding='utf-8') as f:
            return json.load(f)
    
    def on_parameter_changed(self, field: str, value):
        """किसी भी इनपुट परिवर्तन पर UI सिग्नल द्वारा ट्रिगर"""
        self._state[field] = value
        self.save_state(self._state)
```

### 7. समवर्ती थ्रेड आर्किटेक्चर और सिग्नल डिस्पैच

* **सिग्नल-आधारित इवेंट डिस्पैच:** सभी इंटर-मॉड्यूल संचार एक **टाइप्ड Signal/Slot पैटर्न** का उपयोग करता है — वर्कर थ्रेड टाइप्ड सिग्नल (जैसे, `price_updated`, `error_occurred`) एमिट करते हैं जिन्हें UI थ्रेड थ्रेड सीमाओं के पार सुरक्षित रूप से प्राप्त करता है।
* **समवर्ती वर्कर पूल:** प्रत्येक एक्सचेंज का बिडिंग स्कैनर कॉन्फ़िगर करने योग्य समवर्तिता के साथ एक **समर्पित थ्रेड पूल एक्सीक्यूटर** में चलता है। यदि API त्रुटियाँ लौटाता है, तो वर्कर आगे की विफलताओं को रोकने के लिए **कूलडाउन पॉलिसी** (प्रति-विज्ञापन या वैश्विक) लागू करता है।
* **ग्रेसफुल डिग्रेडेशन:** निरंतर API त्रुटियों पर, सिस्टम एक **कूलडाउन मैकेनिज़्म** लागू करता है — किसी एकल विज्ञापन पर 3 लगातार विफलताओं के बाद, वह विज्ञापन 5-मिनट की कूलडाउन विंडो में प्रवेश करता है जबकि अन्य विज्ञापन सामान्य रूप से कार्य करते रहते हैं।

#### 💡 कार्यान्वयन अवधारणा (थ्रेड आर्किटेक्चर)
```python
# [स्यूडोकोड] कूलडाउन सुरक्षा के साथ सिग्नल-आधारित वर्कर
from qt_core import QObject, Signal, QTimer
from concurrent.futures import ThreadPoolExecutor

class BiddingWorker(QObject):
    # थ्रेड-सेफ UI संचार के लिए टाइप्ड सिग्नल
    price_updated = Signal(str, float)      # (ad_id, new_price)
    error_occurred = Signal(str, str)       # (ad_id, error_message)
    cycle_completed = Signal(str)           # (summary)
    
    def __init__(self, api_client, configs):
        super().__init__()
        self._api = api_client
        self._configs = configs
        self._executor = ThreadPoolExecutor(max_workers=20)
        self._timer = QTimer()
        self._timer.setInterval(2000)  # 2-सेकंड पोलिंग चक्र
        self._timer.timeout.connect(self._poll)
    
    def start(self):
        self._timer.start()
    
    def _poll(self):
        """बिडिंग चक्र को थ्रेड पूल में सबमिट करें"""
        self._executor.submit(self._run_cycle)
    
    def _run_cycle(self):
        """सभी सक्रिय विज्ञापनों को समवर्ती रूप से प्रोसेस करें"""
        active = [c for c in self._configs if c.is_active and not c.is_in_cooldown()]
        for config in active:
            try:
                result = self._process_ad(config)
                if result.success:
                    self.price_updated.emit(config.ad_id, result.new_price)
            except APIError as e:
                config.increment_error(str(e))
                self.error_occurred.emit(config.ad_id, str(e))
    
    def stop(self):
        self._timer.stop()
        self._executor.shutdown(wait=False)
```

### API प्रमाणीकरण और रिक्वेस्ट साइनिंग
```python
# [स्यूडोकोड] मल्टी-एक्सचेंज प्रमाणीकरण — एकीकृत साइनिंग इंटरफ़ेस
class UnifiedAuthenticator:
    """प्लेटफ़ॉर्म-विशिष्ट अनुकूलन के साथ HMAC-SHA256 साइनिंग संभालता है"""
    
    def sign_binance(self, params: dict) -> dict:
        # Binance: URL क्वेरी स्ट्रिंग पर सिग्नेचर
        params['timestamp'] = current_time_ms()
        query = urlencode(sorted(params.items()))
        params['signature'] = hmac_sha256(self.secret, query)
        return params
    
    def sign_bybit(self, payload: str) -> dict:
        # Bybit: हेडर-कंपोज़्ड स्ट्रिंग पर सिग्नेचर
        ts = str(current_time_ms())
        sign_str = ts + self.api_key + str(self.recv_window) + payload
        return {"X-BAPI-SIGN": hmac_sha256(self.secret, sign_str)}
    
    def sign_okx(self, method: str, path: str, body: str) -> dict:
        # OKX: रिक्वेस्ट घटकों पर Base64-एनकोडेड HMAC
        ts = iso_timestamp()
        sign_str = ts + method.upper() + path + body
        signature = base64_encode(hmac_sha256(self.secret, sign_str))
        return {"OK-ACCESS-SIGN": signature, "OK-ACCESS-PASSPHRASE": self.passphrase}
```

## 💡 अक्सर पूछे जाने वाले प्रश्न (Q&A)

**प्रश्न: binance-p2p-bot क्या है और यह क्या करता है?**
उत्तर: `binance-p2p-bot` **Binance, Bybit, और OKX** पर P2P व्यापारियों के लिए एक पेशेवर-स्तर का डेस्कटॉप ऑटोमेशन क्लाइंट है। यह प्रतिस्पर्धी डेटा के आधार पर आपके USDT विज्ञापन मूल्यों को स्वचालित रूप से समायोजित करता है, कई एक्सचेंजों पर स्टेबलकॉइन विज्ञापनों का प्रबंधन करता है, और इन्वेंट्री की निगरानी करता है — सब कुछ सुरक्षित स्थानीय API कनेक्शन के माध्यम से।

**प्रश्न: binance-p2p-bot ऑटो-बिडिंग एल्गोरिदम कैसे काम करता है?**
उत्तर: `binance-p2p-bot` समय-समय पर REST API के माध्यम से प्रतिस्पर्धी विज्ञापन सूचियों को पोल करता है और इष्टतम काउंटर-प्राइस की गणना के लिए एक **वेटेड स्कोरिंग एल्गोरिदम** लागू करता है। जब कोई प्रतिस्पर्धी अपनी कीमत बदलता है, तो बॉट आपकी निर्धारित सुरक्षित रेंज (न्यूनतम/अधिकतम मूल्य सीमाओं) के भीतर प्रतिक्रिया की गणना करता है और एक्सचेंज API के माध्यम से आपके विज्ञापन को अपडेट करता है — जिससे आपको अपने लाभ मार्जिन से बाहर बोली लगाए बिना प्रतिस्पर्धी स्थिति बनाए रखने में मदद मिलती है।

**प्रश्न: क्या binance-p2p-bot कई एक्सचेंजों का समर्थन करता है?**
उत्तर: हाँ। यह USDT P2P बॉट मूल रूप से **Binance**, **Bybit**, और **OKX** P2P मार्केट्स का समर्थन करता है। प्रत्येक एक्सचेंज अपने स्वयं के थ्रेड-आइसोलेटेड मॉड्यूल में एक समर्पित एडैप्टर के साथ चलता है, जो प्लेटफ़ॉर्म-विशिष्ट अंतरों (जैसे OKX की सख्त रेट लिमिट और Bybit की ऑनलाइन-बिफ़ोर-एडिट आवश्यकता) को पारदर्शी रूप से संभालता है।

**प्रश्न: क्या USDT P2P बॉट API रेट लिमिट को सुरक्षित रूप से संभालता है?**
उत्तर: हाँ। `binance-p2p-bot` एक **बहु-स्तरीय सुरक्षा प्रणाली** लागू करता है: (1) तेज़ डुप्लिकेट अनुरोधों को फ़िल्टर करने के लिए इनपुट डिबाउंस, (2) लगातार त्रुटियों के बाद प्रति-विज्ञापन कूलडाउन, और (3) एक वैश्विक कूलडाउन मैकेनिज़्म जो API विफलताएं जमा होने पर ऑपरेशन को रोकता है। OKX की 5-कॉल/मिनट सीमा को टाइमर-आधारित रिक्वेस्ट स्पेसिंग वाली एक समर्पित डिबाउंस कतार द्वारा संभाला जाता है।

**प्रश्न: binance-p2p-bot किन क्रिप्टोकरेंसी और फ़िएट मुद्राओं का समर्थन करता है?**
उत्तर: `binance-p2p-bot` Binance, Bybit, और OKX P2P मार्केटप्लेस पर उपलब्ध सभी क्रिप्टो एसेट्स और फ़िएट मुद्राओं के साथ काम करता है — जिसमें USDT, USDC, FDUSD, BTC, ETH, और दर्जनों फ़िएट गेटवे (USD, EUR, PHP, INR, RUB, TRY, THB, VND, IDR, ARS, VES, PKR, और अधिक) शामिल हैं।

**प्रश्न: ऑटो इन्वेंट्री रिप्लेनिशमेंट कैसे काम करता है?**
उत्तर: `binance-p2p-bot` आपके फ़ंडिंग अकाउंट बैलेंस की निगरानी करता है और, जब परिवर्तनीय एसेट्स (USDC, FDUSD) एक कॉन्फ़िगर करने योग्य थ्रेशोल्ड से अधिक हो जाती हैं, तो स्वचालित रूप से एक **मल्टी-स्टेप कन्वर्ज़न पाइपलाइन** निष्पादित करता है — स्पॉट में ट्रांसफ़र, USDT के लिए बिक्री, और फ़ंडिंग में वापस ट्रांसफ़र। अपडेटेड USDT बैलेंस फिर सभी सक्रिय विज्ञापनों में सिंक्रनाइज़ किया जाता है।

**प्रश्न: क्या इस binance-p2p-bot के साथ मेरी API कुंजी सुरक्षित है?**
उत्तर: हाँ। सभी API क्रेडेंशियल आपकी मशीन पर स्थानीय रूप से संग्रहीत होते हैं। बॉट पूरी तरह से आपके स्थानीय डिवाइस पर शून्य बाहरी सर्वर निर्भरता के साथ कार्य करता है — कोई तृतीय-पक्ष रिले या क्लाउड स्टोरेज शामिल नहीं है। विज़ुअल सुरक्षा के लिए API कुंजियाँ UI में मास्क की जाती हैं। हम आपकी API कुंजियों को कभी भी "Withdrawal" अनुमतियाँ नहीं देने की सलाह देते हैं।

**प्रश्न: यदि मैं अपना कंप्यूटर रीस्टार्ट करता हूँ, तो क्या मैं अपनी कॉन्फ़िगरेशन खो दूँगा?**
उत्तर: नहीं। `binance-p2p-bot` में **ऑटो-सेव पर्सिस्टेंस** की सुविधा है — सभी UI सेटिंग्स, विज्ञापन कॉन्फ़िगरेशन, मूल्य सीमाएं, और बिडिंग पैरामीटर प्रत्येक परिवर्तन पर एक स्थानीय JSON फ़ाइल में लिखे जाते हैं। रीस्टार्ट पर, बॉट स्वचालित रूप से आपकी सटीक कॉन्फ़िगरेशन बहाल करता है।

**प्रश्न: binance-p2p-bot किस तकनीकी स्टैक का उपयोग करता है?**
उत्तर: बॉट **Python** और **Qt6** के साथ बनाया गया है, जो एक नेटिव डेस्कटॉप अनुभव प्रदान करता है। यह थ्रेड-सेफ इवेंट डिस्पैच के लिए Signal/Slot पैटर्न, API पोलिंग के लिए समवर्ती थ्रेड पूल, API प्रमाणीकरण के लिए HMAC-SHA256, और कॉन्फ़िगरेशन पर्सिस्टेंस के लिए JSON-आधारित ऑटो-सेव का उपयोग करता है। कोई बाहरी सर्वर, डेटाबेस, या क्लाउड सेवाओं की आवश्यकता नहीं है।

**प्रश्न: क्या मैं प्रतिबंधित नेटवर्क वातावरण में binance-p2p-bot चला सकता हूँ?**
उत्तर: हाँ। बॉट पूरी तरह से आपकी स्थानीय मशीन पर कार्य करता है। सभी API कनेक्शन सीधे आपके डिवाइस से एक्सचेंज सर्वरों पर जाते हैं — कोई तृतीय-पक्ष रिले शामिल नहीं है। जटिल नेटवर्क स्थितियों वाले क्षेत्रों के उपयोगकर्ताओं के लिए, मानक सिस्टम-स्तरीय प्रॉक्सी या VPN कॉन्फ़िगरेशन OS स्तर पर लागू किए जा सकते हैं।

**प्रश्न: P2P में सेल और बाय बिडिंग में क्या अंतर है?**
उत्तर: `binance-p2p-bot` में, **Sell USDT** का अर्थ है कि आप USDT बेच रहे हैं और फ़िएट प्राप्त कर रहे हैं — बॉट खरीदारों के लिए प्रतिस्पर्धी मूल्य पेश करने के लिए प्रतिस्पर्धा करता है। **Buy USDT** का अर्थ है कि आप फ़िएट से USDT खरीद रहे हैं — बॉट विक्रेताओं को आकर्षित करने के लिए प्रतिस्पर्धा करता है। प्रत्येक दिशा में स्वतंत्र मूल्य सीमाएं, ऑफ़सेट मान, और प्रतिस्पर्धी फ़िल्टरिंग कॉन्फ़िगरेशन होती हैं।
