**binance-p2p-bot** ایک پیشہ ورانہ درجے کا ڈیسک ٹاپ کلائنٹ ہے جو **Binance**، **Bybit**، اور **OKX** پر کام کرنے والے **P2P تاجروں (اشتہار دہندگان)** کے لیے ڈیزائن کیا گیا ہے۔ دستی مارکیٹ مانیٹرنگ سے API سے چلنے والے پروگرامیٹک عمل میں منتقلی کے ذریعے، یہ USDT P2P بوٹ آپ کو تینوں بڑے ایکسچینجز پر بیک وقت مسلسل قیمت فوائد اور موثر آرڈر پروسیسنگ رفتار برقرار رکھنے میں مدد کرتا ہے۔

**ماڈیولر ایونٹ ڈریون آرکیٹیکچر** اور **شیڈولڈ API پولنگ اور قابل ترتیب وقفوں** پر بنایا گیا، یہ دستی ٹریکنگ میں تاخیر، آربیٹراژرز کو آمدنی کے نقصان، اور تھکا دینے والے "سکرین ٹریپ" کے مسئلے کو حل کرتا ہے۔ `binance-p2p-bot` انجن موجودہ مارکیٹ حالات کی بنیاد پر پولنگ فریکوئنسیز کو ایڈجسٹ کرنے کے لیے **اڈاپٹیو انٹروَل شیڈولنگ** استعمال کرتا ہے۔

## 🚀 بنیادی خصوصیات (الگورتھم اور ورک فلو)

### 1. آٹو بِڈنگ اور ریئل ٹائم قیمت ایڈجسٹمنٹ (سمارٹ رینک سنائپنگ)

* **شیڈولڈ مارکیٹ پولنگ:** `binance-p2p-bot` انجن ہر ایکسچینج کی P2P اشتہار فہرست کے خلاف وقتاً فوقتاً REST API استفسارات انجام دیتا ہے، ہر پولنگ سائیکل میں حریف قیمت تبدیلیوں کا پتہ لگاتا ہے۔
* **ویٹڈ پرائیرٹی پرائسنگ:** ایک **ویٹڈ سکورنگ الگورتھم** استعمال کرتا ہے جو حریف قیمتوں کا جائزہ لیتا ہے، قابل ترتیب آفسیٹ ویلیوز (مثلاً $0.01 ایج) لاگو کرتا ہے، اور سرکردہ اشتہار پوزیشن برقرار رکھنے کے لیے بہترین جوابی قیمت کا حساب لگاتا ہے۔
* **ملٹی کرائٹیریا حریف فلٹرنگ:** ایک **قابل ترتیب فلٹر پائپ لائن** تعینات کرتا ہے — انجن حریف اشتہارات کا تجارتی رقم کی حدود، کم از کم آرڈر حدود، ادائیگی کے طریقے کی مطابقت، اور آن لائن حیثیت کے ذریعے جائزہ لیتا ہے تاکہ متعلقہ مقابلہ شناخت کیا جا سکے۔
* **محفوظ رینج پروٹیکشن:** انتہائی فیاٹ/کرپٹو اتار چڑھاؤ کے دوران غیر منافع بخش لین دین کو روکنے کے لیے **قابل ترتیب حد بندی قواعد** کے ذریعے سخت قیمت کی چھت اور فرش نافذ کرتا ہے۔

#### 💡 عمل درآمد کا تصور (سمارٹ سنائپنگ)
```python
# [سیوڈوکوڈ] ویٹڈ پرائیرٹی پرائسنگ انجن
class PricingEngine:
    def __init__(self, exchange_adapter, config):
        self.adapter = exchange_adapter
        self.config = config
        self.poll_interval_ms = 2000
    
    async def on_poll_cycle(self):
        """شیڈولڈ پولنگ ٹائمر سے ٹرگر ہوتا ہے"""
        # REST API کے ذریعے موجودہ حریف اشتہار فہرست حاصل کریں
        competitor_ads = await self.adapter.search_ads(
            asset=self.config.asset,
            fiat=self.config.fiat,
            trade_type=self.config.trade_type
        )
        
        # ملٹی کرائٹیریا فلٹر پائپ لائن لاگو کریں
        candidates = self.filter_pipeline.execute(
            competitor_ads,
            filters=self.config.active_filters
        )
        
        # ویٹڈ سکورنگ کے ذریعے ہدف قیمت کا حساب لگائیں
        target_price = self.scoring_engine.calculate(
            candidates,
            offset=self.config.price_offset,
            floor=self.config.min_price,
            ceiling=self.config.max_price
        )
        
        # اگر تبدیلی ہو تو قیمت اپ ڈیٹ عمل میں لائیں
        if abs(target_price - self.current_price) > self.config.epsilon:
            await self.adapter.update_ad_price(self.ad_id, target_price)
            self.current_price = target_price
```

### 2. ملٹی ایکسچینج یونیفائیڈ کنٹرول (Binance + Bybit + OKX)

* **تھریڈ آئسولیٹڈ ایکسچینج ماڈیولز:** ہر ایکسچینج (Binance، Bybit، OKX) اپنے **آزاد تھریڈ پول** میں الگ شیڈولنگ کے ساتھ چلتا ہے، کراس ایکسچینج مداخلت کو روکتا ہے۔
* **ایبسٹریکٹ اڈاپٹر پیٹرن:** ایک یونیفائیڈ `ExchangeAdapterFactory` ایک **فیکٹری پیٹرن** کے ذریعے پلیٹ فارم مخصوص اڈاپٹرز بناتا ہے، تینوں ایکسچینجز میں API فرق کو شفافانہ طور پر معمول پر لاتا ہے۔
* **OKX ریٹ لمٹ قطار:** OKX کی سخت 5 کالز/منٹ حد کو **ٹائمر پر مبنی وقفوں والی ڈیباؤنس قطار** کے ذریعے سنبھالا جاتا ہے، جو حدود میں رہنے کے لیے بیک وقت اپ ڈیٹ درخواستوں کو خودکار طور پر یکجا کرتی ہے۔

#### 💡 عمل درآمد کا تصور (ملٹی ایکسچینج اڈاپٹر)
```python
# [سیوڈوکوڈ] تھریڈ آئسولیٹڈ ایکسچینج اڈاپٹر فیکٹری
class ExchangeAdapterFactory:
    _registry = {}
    
    @classmethod
    def register(cls, platform: str, adapter_cls):
        cls._registry[platform] = adapter_cls
    
    @classmethod
    def create(cls, platform: str, credentials: dict):
        adapter_cls = cls._registry.get(platform)
        if not adapter_cls:
            raise UnsupportedPlatformError(platform)
        
        # ہر اڈاپٹر اپنے تھریڈ پول میں چلتا ہے
        thread_pool = ThreadPoolExecutor(
            max_workers=platform_config(platform).max_concurrency
        )
        return adapter_cls(credentials, executor=thread_pool)

# تمام معاون ایکسچینجز کو رجسٹر کریں
ExchangeAdapterFactory.register("binance", BinanceP2PAdapter)
ExchangeAdapterFactory.register("bybit", BybitP2PAdapter)
ExchangeAdapterFactory.register("okx", OkxP2PAdapter)
```

### 3. خودکار انوینٹری پائپ لائن اور آٹو ری پلینشمنٹ

* **ملٹی سٹیپ اثاثہ تبدیلی:** `binance-p2p-bot` فنڈنگ اکاؤنٹ بیلنسز کی نگرانی کرتا ہے اور خودکار طور پر ایک **ترتیب وار تبدیلی پائپ لائن** (USDC/FDUSD → اسپاٹ مارکیٹ سیل → USDT → فنڈنگ اکاؤنٹ) ٹرگر کرتا ہے، آپ کی USDT انوینٹری کو بھرا رکھتا ہے۔
* **تھریشولڈ پر مبنی نگرانی:** **قابل ترتیب بیلنس تھریشولڈز** استعمال کرتا ہے تاکہ یہ پتہ لگایا جا سکے کہ قابل تبدیل اثاثے کم از کم رقم سے تجاوز کر گئے ہیں، صرف اسی وقت تبدیلی کے چکر ٹرگر کرتا ہے جب بامعنی بیلنسز دستیاب ہوں۔
* **بیچ انوینٹری سنکرونائزیشن:** تبدیلی کے بعد، بوٹ دستیاب USDT کا دوبارہ حساب لگاتا ہے اور ایک ہی بیچ آپریشن میں تمام فعال اشتہارات پر **اپ ڈیٹ شدہ انوینٹری نشر** کرتا ہے، تمام فہرستوں میں مستقل مقداریں یقینی بناتا ہے۔

#### 💡 عمل درآمد کا تصور (انوینٹری پائپ لائن)
```python
# [سیوڈوکوڈ] ملٹی سٹیپ اثاثہ تبدیلی پائپ لائن
class InventoryPipeline:
    CONVERSION_ROUTES = {
        "USDC": ["transfer_to_spot", "market_sell_for_usdt", "transfer_to_funding"],
        "FDUSD": ["transfer_to_spot", "market_sell_for_usdt", "transfer_to_funding"],
    }
    
    async def run_replenishment_cycle(self):
        """تھریشولڈ سے ٹرگر شدہ انوینٹری ری پلینشمنٹ"""
        # 1. فنڈنگ اکاؤنٹ میں قابل تبدیل اثاثوں کی جانچ کریں
        balances = await self.exchange.get_funding_balances()
        
        for asset, amount in balances.items():
            if asset in self.CONVERSION_ROUTES and amount >= self.min_threshold:
                # 2. ترتیب وار تبدیلی مراحل عمل میں لائیں
                route = self.CONVERSION_ROUTES[asset]
                for step in route:
                    await self.execute_step(step, asset, amount)
        
        # 3. حتمی USDT بیلنس کا دوبارہ حساب لگائیں
        usdt_available = await self.exchange.get_funding_balance("USDT")
        
        # 4. تمام فعال اشتہارات پر اپ ڈیٹ شدہ انوینٹری نشر کریں
        for ad in self.active_ads:
            current = await self.adapter.get_ad_detail(ad.id)
            new_inventory = self.calculate_new_inventory(current, usdt_available)
            if abs(new_inventory - current.inventory) >= 1.0:
                await self.adapter.update_ad_inventory(ad.id, new_inventory)
```

### 4. حریف تجزیہ اور بلیک لسٹ انجن

* **ملٹی اسٹیج فلٹر پائپ لائن:** انجن حریف اشتہارات پر ایک **ترتیب وار فلٹر چین** لاگو کرتا ہے — قیمت کی رینج، تجارتی رقم کی حدود، ادائیگی کے طریقے کی اوورلیپ، آن لائن حیثیت، اور شناختی جانچ کا جائزہ لے کر ایک بہتر امیدوار فہرست تیار کرتا ہے۔
* **قابل ترتیب بلیک لسٹ:** `binance-p2p-bot` ایک **فی اشتہار بلیک لسٹ** کی حمایت کرتا ہے جہاں تاجر مخصوص حریف اشتہار نمبروں کو خارج کر سکتے ہیں۔ بلیک لسٹ شدہ اشتہارات کو ہر بِڈنگ سائیکل میں خودکار طور پر چھوڑ دیا جاتا ہے۔
* **ادائیگی طریقہ مماثلت:** ادائیگی کے طریقے کی صفوں پر **سیٹ انٹرسیکشن لاجک** استعمال کرتا ہے تاکہ یقینی بنایا جا سکے کہ بوٹ صرف ان اشتہار دہندگان کے خلاف مقابلہ کرے جو آپ کے اشتہار کے ساتھ کم از کم ایک مشترک ادائیگی چینل شریک کرتے ہیں۔

#### 💡 عمل درآمد کا تصور (حریف فلٹر)
```python
# [سیوڈوکوڈ] ملٹی اسٹیج حریف فلٹر پائپ لائن
class CompetitorFilterPipeline:
    def __init__(self, config: AdConfig):
        self.config = config
        self.filters = [
            self.filter_self,
            self.filter_blacklist,
            self.filter_price_range,
            self.filter_trade_amount,
            self.filter_min_order,
            self.filter_payment_methods,
            self.filter_online_status,
        ]
    
    def execute(self, raw_ads: list) -> list:
        """فلٹر چین کو ترتیب وار لاگو کریں"""
        candidates = raw_ads
        for filter_fn in self.filters:
            candidates = [ad for ad in candidates if filter_fn(ad)]
        return sorted(candidates, key=lambda ad: ad.price)
    
    def filter_blacklist(self, ad) -> bool:
        return ad.ad_id not in self.config.blacklist
    
    def filter_payment_methods(self, ad) -> bool:
        """سیٹ انٹرسیکشن: کم از کم ایک مشترکہ ادائیگی کا طریقہ"""
        my_methods = set(self.config.payment_methods)
        their_methods = set(ad.payment_methods)
        return len(my_methods & their_methods) > 0
    
    def filter_trade_amount(self, ad) -> bool:
        if self.config.max_amount_filter:
            return ad.max_amount >= self.config.max_amount_filter
        return True
```

### 5. اشتہار لائف سائیکل مینجمنٹ اور بلک آپریشنز

* **اسٹیٹس ٹرانزیشن کنٹرول:** ہر اشتہار براہ راست API کالز کے ذریعے اسٹیٹس ٹرانزیشنز (`OFFLINE ↔ ONLINE`) کی حمایت کرتا ہے، غلط آپریشنز کو روکنے کے لیے اسٹیٹس تصدیق کے ساتھ۔
* **ریٹ لمٹ اسپیسنگ کے ساتھ بلک ٹوگل:** برسٹ ٹرگرڈ ریٹ لمٹس کو روکنے کے لیے قابل ترتیب API اسپیسنگ کے ساتھ تمام اشتہارات کے لیے ایک کلک فعال/غیر فعال۔
* **ویٹڈ ایوریج پرائس ٹریکنگ:** C2C آرڈر ہسٹری سے حساب شدہ ویٹڈ ایوریج خرید/فروخت قیمتوں سمیت لائیو میٹرکس، جو تاجروں کو ریئل ٹائم منافع بصیرت فراہم کرتا ہے۔

#### 💡 عمل درآمد کا تصور (اشتہار اسٹیٹس مینیجر)
```python
# [سیوڈوکوڈ] بلک آپریشنز کے ساتھ اشتہار اسٹیٹس مینیجر
class AdStatusManager:
    async def set_status(self, ad_id: str, online: bool):
        """API کے ذریعے واحد اشتہار اسٹیٹس ٹوگل کریں"""
        current = await self.adapter.get_ad_detail(ad_id)
        target_online = online
        
        if current.is_online == target_online:
            return  # پہلے سے مطلوبہ حالت میں ہے
        
        await self.adapter.set_ad_status(ad_id, online=target_online)
        logger.info(f"اشتہار {ad_id}: {'ONLINE' if target_online else 'OFFLINE'}")
    
    async def bulk_toggle(self, ad_ids: list, online: bool):
        """API اسپیسنگ کے ساتھ بیچ اسٹیٹس تبدیلی"""
        for ad_id in ad_ids:
            await self.set_status(ad_id, online)
            await asyncio.sleep(0.5)  # ریٹ لمٹ اسپیسنگ
```

```python
class ConfigurationManager:
    def __init__(self, config_path: str):
        self.config_path = config_path
        self._state = {}
    
    def save_state(self, ui_state: dict):
        """ہر پیرامیٹر تبدیلی پر ایٹامک رائٹ"""
        data = {
            "api_key": ui_state.get("api_key"),
            "api_secret": ui_state.get("api_secret"),
            "language": ui_state.get("language", "en"),
            "sell_ads": ui_state.get("sell_ads", []),
            "buy_ads": ui_state.get("buy_ads", []),
            "replenish_enabled": ui_state.get("replenish_enabled", False),
        }
        # عارضی فائل + نام تبدیلی کے ذریعے ایٹامک رائٹ
        tmp_path = self.config_path + ".tmp"
        with open(tmp_path, 'w', encoding='utf-8') as f:
            json.dump(data, f, indent=2, ensure_ascii=False)
        os.replace(tmp_path, self.config_path)
    
    def load_state(self) -> dict:
        """ڈسک سے مکمل کنفیگریشن بحال کریں"""
        if not os.path.exists(self.config_path):
            return self._default_config()
        with open(self.config_path, 'r', encoding='utf-8') as f:
            return json.load(f)
    
    def on_parameter_changed(self, field: str, value):
        """کسی بھی ان پٹ تبدیلی پر UI سگنل سے ٹرگر ہوتا ہے"""
        self._state[field] = value
        self.save_state(self._state)
```

### 7. ہم وقت تھریڈ آرکیٹیکچر اور سگنل ڈسپیچ

* **سگنل پر مبنی ایونٹ ڈسپیچ:** تمام بین ماڈیول مواصلات ایک **ٹائپڈ Signal/Slot پیٹرن** استعمال کرتا ہے — ورکر تھریڈز ٹائپڈ سگنلز (مثلاً `price_updated`، `error_occurred`) ایمیٹ کرتے ہیں جنہیں UI تھریڈ تھریڈ حدود کے پار محفوظ طریقے سے وصول کرتا ہے۔
* **ہم وقت ورکر پول:** ہر ایکسچینج کا بِڈنگ سکینر قابل ترتیب ہم وقتی کے ساتھ ایک **مخصوص تھریڈ پول ایگزیکیوٹر** میں چلتا ہے۔ اگر API غلطیاں واپس کرتا ہے، تو ورکر مزید ناکامیوں کو روکنے کے لیے **کولڈاؤن پالیسیاں** (فی اشتہار یا عالمی) لاگو کرتا ہے۔
* **گریس فُل ڈیگریڈیشن:** مسلسل API غلطیوں پر، سسٹم ایک **کولڈاؤن میکانزم** لاگو کرتا ہے — کسی واحد اشتہار پر 3 مسلسل ناکامیوں کے بعد، وہ اشتہار 5 منٹ کی کولڈاؤن ونڈو میں داخل ہوتا ہے جبکہ دیگر اشتہارات عام طور پر کام کرتے رہتے ہیں۔

#### 💡 عمل درآمد کا تصور (تھریڈ آرکیٹیکچر)
```python
# [سیوڈوکوڈ] کولڈاؤن پروٹیکشن کے ساتھ سگنل پر مبنی ورکر
from qt_core import QObject, Signal, QTimer
from concurrent.futures import ThreadPoolExecutor

class BiddingWorker(QObject):
    # تھریڈ سیف UI مواصلات کے لیے ٹائپڈ سگنلز
    price_updated = Signal(str, float)      # (ad_id, new_price)
    error_occurred = Signal(str, str)       # (ad_id, error_message)
    cycle_completed = Signal(str)           # (summary)
    
    def __init__(self, api_client, configs):
        super().__init__()
        self._api = api_client
        self._configs = configs
        self._executor = ThreadPoolExecutor(max_workers=20)
        self._timer = QTimer()
        self._timer.setInterval(2000)  # 2 سیکنڈ پولنگ سائیکل
        self._timer.timeout.connect(self._poll)
    
    def start(self):
        self._timer.start()
    
    def _poll(self):
        """بِڈنگ سائیکل کو تھریڈ پول میں جمع کرائیں"""
        self._executor.submit(self._run_cycle)
    
    def _run_cycle(self):
        """تمام فعال اشتہارات کو ہم وقت پروسیس کریں"""
        active = [c for c in self._configs if c.is_active and not c.is_in_cooldown()]
        for config in active:
            try:
                result = self._process_ad(config)
                if result.success:
                    self.price_updated.emit(config.ad_id, result.new_price)
            except APIError as e:
                config.increment_error(str(e))
                self.error_occurred.emit(config.ad_id, str(e))
    
    def stop(self):
        self._timer.stop()
        self._executor.shutdown(wait=False)
```

```python
# [سیوڈوکوڈ] ملٹی ایکسچینج توثیق — یونیفائیڈ سائننگ انٹرفیس
class UnifiedAuthenticator:
    """پلیٹ فارم مخصوص ایڈاپٹیشنز کے ساتھ HMAC-SHA256 سائننگ کو سنبھالتا ہے"""
    
    def sign_binance(self, params: dict) -> dict:
        # Binance: URL کوئری سٹرنگ پر دستخط
        params['timestamp'] = current_time_ms()
        query = urlencode(sorted(params.items()))
        params['signature'] = hmac_sha256(self.secret, query)
        return params
    
    def sign_bybit(self, payload: str) -> dict:
        # Bybit: ہیڈر سے مرتب سٹرنگ پر دستخط
        ts = str(current_time_ms())
        sign_str = ts + self.api_key + str(self.recv_window) + payload
        return {"X-BAPI-SIGN": hmac_sha256(self.secret, sign_str)}
    
    def sign_okx(self, method: str, path: str, body: str) -> dict:
        # OKX: ریکوئسٹ اجزاء پر Base64 انکوڈڈ HMAC
        ts = iso_timestamp()
        sign_str = ts + method.upper() + path + body
        signature = base64_encode(hmac_sha256(self.secret, sign_str))
        return {"OK-ACCESS-SIGN": signature, "OK-ACCESS-PASSPHRASE": self.passphrase}
```

جواب: بوٹ **Python** اور **Qt6** کے ساتھ بنایا گیا ہے، جو ایک نیٹو ڈیسک ٹاپ تجربہ فراہم کرتا ہے۔ یہ تھریڈ سیف ایونٹ ڈسپیچ کے لیے Signal/Slot پیٹرنز، API پولنگ کے لیے ہم وقت تھریڈ پولز، API توثیق کے لیے HMAC-SHA256، اور کنفیگریشن پرسسٹنس کے لیے JSON پر مبنی آٹو سیو استعمال کرتا ہے۔ کسی بیرونی سرور، ڈیٹابیس، یا کلاؤڈ سروسز کی ضرورت نہیں ہے۔
